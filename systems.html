Advanced • Backend / Networking

[Matchmaking System]

A scalable matchmaking system with party support. Players can queue together and are guaranteed to enter the same arena. Party leadership is reassigned automatically if the leader disconnects. Built using decoupled services to separate party state management from arena allocation. 


-- Party creation
function PartyService:CreateParty(player)
    local Party = {
        Id = player.UserId,
        Leader = player,
        Members = {player},
        State = "Idle"
    }
    self.Parties[player.UserId] = Party
    return Party
end

-- Prioritise filling half-empty servers first
function ArenaService:FindPlaceForParty()
    local emptyPlace = nil
    for _, place in ipairs(placesFolder:GetChildren()) do
        local parties = place:GetAttribute("Parties") or 0
        if parties == 1 then
            return place
        end
        if parties == 0 and not emptyPlace then
            emptyPlace = place
        end
    end
    return emptyPlace
end

Advanced • Client / Validation

[Building System]

A grid-based building system with real-time ghost previews, collision checks, and server-authoritative placement validation. Supports both desktop and mobile input. Structures snap cleanly to configurable grid tiles while occupied spaces are rejected to prevent overlap. 


local snapPos = Vector3.new(

    BuildArea.Position.Y + GRID_SIZE/2,
    math.floor(rawPos.Z/GRID_SIZE + 0.5) * GRID_SIZE
)

local function tileFree(pos)
    for _, defense in ipairs(Defenses:GetChildren()) do
        if defense.PrimaryPart then
            local p = defense.PrimaryPart.Position
            if (Vector3.new(p.X,0,p.Z) - Vector3.new(pos.X,0,pos.Z)).Magnitude < GRID_SIZE then
                return false
            end
        end
    end
    return true
end

Intermediate • UI / Animation

[Notification System]

A stacked notification framework that queues multiple announcements and smoothly repositions active notifications when the stack changes. Uses a NumberValue-driven tween to animate UIGradient transparency for wipe animations that TweenService cannot perform directly. 


local function reposition(animate)
    for index, frame in ipairs(Active) do
        local targetPos = UDim2.new(
            frame.Position.X.Scale,
            frame.Position.X.Offset,
            0,
            (index - 1) * SlotHeight
        )

        if animate then
            TweenService:Create(frame, SlideTweenInfo, {
                Position = targetPos
            }):Play()
        end
    end
end

[Settings Framework]

A data-driven client-side settings framework supporting toggles and draggable sliders. Settings are applied instantly via player attributes, allowing any system to react without remote events. New settings can be added through UI attributes alone without changing code. 


local function setUpOffOn(option, frame, knob)
    local isOn = false

    local function update()
        knob.Position = isOn
            and UDim2.new(0.82,0,0,0)
            or UDim2.new(0,0,0,0)

        player:SetAttribute(option.Name, isOn)
    end
end
